# DPLL SAT Solver на Kotlin

Реализация классического алгоритма DPLL (Davis-Putnam-Logemann-Loveland) на языке Kotlin. Проект представляет собой решатель задачи о выполнимости булевых формул (SAT) для формул в конъюнктивной нормальной форме (КНФ).

![Kotlin](https://img.shields.io/badge/kotlin-%237F52FF.svg?style=for-the-badge&logo=kotlin&logoColor=white)

## Об алгоритме DPLL

DPLL — это рекурсивный алгоритм с возвратами (бэктрекинг), который эффективно определяет, имеет ли заданная булева формула хотя бы одно решение.

Основная идея состоит в итеративном упрощении формулы с помощью детерминированных правил. Когда дальнейшее упрощение невозможно, алгоритм делает предположение о значении одной из переменных и рекурсивно проверяет, ведет ли это предположение к решению.

В данной реализации используется эвристика **MOMs (Maximum Occurrences in clauses of Minimum size)**.

**Как это работает:**
1. Находится минимальная длина дизъюнкта в текущей формуле (длиной >= 2).
2. Рассматриваются только дизъюнкты этой минимальной длины.
3. Подсчитывается, какой литерал встречается в них чаще всего.
4. Переменная, соответствующая этому литералу, выбирается для следующего ветвления.

**Обоснование:** Короткие дизъюнкты наиболее ограничены — их легче всего сделать ложными. Выбирая переменную из такого дизъюнкта, мы с большей вероятностью либо быстро удовлетворим его, либо придем к противоречию, что эффективно сокращает пространство поиска.

## Как запустить

1.  Клонируйте репозиторий:
    ```bash
    git clone https://github.com/LeksinMaksim/SAT-Solver.git
    ```
2.  Запустите программу, передав в качестве аргумента путь к файлу в формате DIMACS CNF.

    Пример:
    ```bash
    java -jar SatSolver.jar path/to/formula.cnf
    ```

### Анализ сложности

В худшем случае задача SAT является NP-полной, и временная сложность DPLL составляет **O(2^n * poly(L))**, где `n` — число переменных, а `L` — общее число литералов. Однако на практике, благодаря правилам упрощения и эффективным эвристикам, таким как MOMs, алгоритм позволяет срезать огромные ветви дерева поиска и работает значительно быстрее.
